## 2024-10-24 - [MahjongTile Keyboard Accessibility]
**Learning:** Custom interactive components built with generic `<div>` tags completely break keyboard navigation and screen reader semantics unless explicitly augmented. In `MahjongTile.tsx`, the `onClick` handler alone resulted in tiles being unselectable via Tab, lacking focus styling, and only reading their image `alt` attributes instead of functioning as a button.
**Action:** When creating custom interactive elements (like the interactive Mahjong tiles in hand), ensure to add `role="button"`, `tabIndex={0}`, an `onKeyDown` handler for 'Enter'/'Space', and `.component:focus-visible` styling globally or modularly. Additionally, providing an `aria-label` ensures accurate semantic announcement beyond the default image alt-text.
